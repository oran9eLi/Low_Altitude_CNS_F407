# 研发协作规则

## 基本原则

1. 需求先冻结，再并行开发。
2. 接口表先冻结，再接线和写驱动。
3. 单模块先验证，再进入主工程。
4. 文档和代码同步提交。
5. 问题必须闭环，禁止口头说明。

## 开发流程

```text
拉取 develop
-> 新建 feature 分支
-> 单模块开发
-> 单模块测试
-> 更新文档
-> 提交合并请求
-> 代码评审
-> 合入 develop
-> 系统联调
-> 阶段发布
```

## 模块进入主工程的条件

1. 接线定义明确。
2. 初始化函数可用。
3. Demo 已单独跑通。
4. 有测试记录。
5. 提供标准 API。
6. 错误状态进入错误码表。
