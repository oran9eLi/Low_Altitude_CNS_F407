# 时钟与频率参考

> 来源：`Core/Src/main.c` → `SystemClock_Config()`
> 开发板：STM32F407 + 8MHz HSE

---

## 一、时钟树

```
  HSE 8MHz
    │
    └→ PLL ──────────→ PLLCLK = 168 MHz
         │  PLLM=8     │
         │  PLLN=336   │
         │  PLLP=/2    │
         │             │
         ├─ /1 → HCLK  = 168 MHz  (AHB: CPU, DMA, SDIO)
         ├─ /4 → PCLK1 =  42 MHz  (APB1: TIM2~7/12~14, USART2/3, UART4/5, SPI2/3, I2C1~3)
         └─ /2 → PCLK2 =  84 MHz  (APB2: TIM1/8~11, USART1/6, SPI1, ADC1~3)

  定时器倍频规则: APBx prescaler ≠ 1 时 → TIMxCLK = 2 × PCLKx
      APB1 prescaler = /4 → TIM2~7 CLK = 2 × 42 = 84 MHz
      APB2 prescaler = /2 → TIM1/8~11 CLK = 2 × 84 = 168 MHz
```

## 三、PWM 频率参数速算

公式：`PWM_freq = TIM_CLK / (PSC + 1) / (ARR + 1)`

### 3.1 电机 PWM (TIM3/4, 84 MHz)

电机驱动通常需要 10~20 kHz PWM（避免人耳可闻啸叫）。

| 目标频率 | PSC | ARR | 分辨率 | 备注 |
|---------|-----|-----|--------|------|
| 10 kHz | 83 | 99 | 100 steps (7 bit) | 粗粒度，不建议 |
| 10 kHz | 0 | 8399 | 8400 steps (13 bit) | **推荐**，高分辨率 |
| 20 kHz | 0 | 4199 | 4200 steps (12 bit) | 高频低噪 |

> 推荐 PSC=0, ARR=8399 → 10 kHz，占空比按 0~8399 写入 CCR。

### 3.2 蜂鸣器 (TIM13, 168 MHz)

蜂鸣器需要可闻频率（2~4 kHz），TIM13 时钟是 TIM3 的 **2 倍**——PSC 不能照搬。

| 目标频率 | PSC | ARR | 说明 |
|---------|-----|-----|------|
| 3 kHz | 83 | 667 | PSC=83 (168M/84=2M基频)，ARR=667 |
| 3 kHz | 0 | 55999 | 全分辨率，ARR 较大 |

> **注意**：TIM13 在 APB2，如果用 TIM3 的 PSC=0, ARR=8399，TIM13 会出 168M/8400 = 20 kHz，人耳可闻范围上限，不一定够低。**TIM13 的 PSC 必须单独算。**

### 3.3 背光 (PB15, GPIO PWM)

LCD_BL 不需要高精度，可用 TIM 输出或 GPIO 软件 PWM。如果用 TIM12（也挂在 APB1, 84 MHz），PSC=84-1, ARR=999-1 → 1 kHz 够用。

## 四、串口波特率参数

公式：`BaudRate = PCLKx / (8 × (2 - OVER8) × USARTDIV)`

| USART | 总线 | 波特率 | OVER8 | USARTDIV | 误差 |
|-------|------|--------|-------|----------|------|
| USART1 | 84 MHz | 115200 | 0 (16×) | 45.57 → 45 + 9/16 | 0% |
| USART2 | 42 MHz | 9600 | 0 (16×) | 273.44 → 273 + 7/16 | 0% |
| USART3 | 42 MHz | 9600 | 0 (16×) | 273.44 → 273 + 7/16 | 0% |
| USART6 | 84 MHz | 115200 | 0 (16×) | 45.57 → 45 + 9/16 | 0% |

> 以上波特率在 HAL 库自动计算后误差均为 0%，无需手动校准。

## 五、I2C 频率

I2C1 挂 MPU6050 + BME280，标准 400 kHz Fast Mode。

```
  PCLK1 = 42 MHz
  I2C 标准模式:  100 kHz → CCR = 42M / (2×100k) = 210
  I2C 快速模式:  400 kHz → CCR = 42M / (3×400k) = 35
```

HAL 库 `I2C_InitTypeDef.ClockSpeed = 400000` 即可自动配置。

## 八、ADC 采样时钟

```
  ADC1 挂在 APB2 (84 MHz)
  ADCCLK 最大 = 36 MHz (PCLK2 / 2 或 /4 /6 /8)
  HAL 默认: ADCCLK = PCLK2 / 2 = 42 MHz → 超限！须手动设为 /4 = 21 MHz
  推荐: PCLK2 / 4 = 21 MHz (ADCPRE = 0b01)
```

## 九、开发注意事项

| 场景 | 要点 |
|------|------|
| 配 TIM3 和 TIM4 | 两定时器同挂 APB1，时钟均为 84 MHz，**PSC/ARR 参数可互相复用** |
| 配 TIM13 蜂鸣器 | TIM13 挂 APB2，时钟 168 MHz = TIM3 的 2 倍，**PSC/ARR 不能照搬 TIM3** |
| 配 USART 波特率 | USART2/3 在 APB1 (42M)，USART1/6 在 APB2 (84M)，波特率寄存器值不同 |
| 配 I2C | I2C1 在 APB1 (42M)，400kHz 快速模式 CCR=35 |
| 配 ADC | GPIO 时钟使能记得 `__HAL_RCC_GPIOx_CLK_ENABLE()`，ADC1 时钟分频 /4 |
| 调 FreeRTOS 延时 | `pdMS_TO_TICKS(ms)` 基于 TIM6 1ms tick，不受 SYSCLK 变更影响 |
| STM32CubeMX 核对 | 用 CubeMX 生成时钟树后，对照本表核对关键频率：HCLK=168, PCLK1=42, PCLK2=84, TIM2~7=84, TIM1/8~11=168 |

## 十、FSMC TFTLCD 屏幕（168 MHz AHB）

### 10.1 时钟源

FSMC 挂在 AHB 总线，时钟 = HCLK = **168 MHz**。一个 HCLK 周期 ≈ 5.95ns。FSMC 用 HCLK 周期数控制 8080 并口读写时序。

### 10.2 时序参数（SRAM/NOR Bank4 模式）

```
  FSMC_BTR 寄存器:
    ADDSET[3:0]  = 地址建立时间 (HCLK 周期)
    DATAST[7:0]  = 数据建立/写脉冲宽度 (HCLK 周期)
    ADDHLD[3:0]  = 地址保持时间

  典型 ILI9341/ILI9488 配置:
    ADDSET = 0x02  → 2 × 6ns ≈ 12ns
    DATAST = 0x04  → 4 × 6ns ≈ 24ns
    BUSTURN = 0x02 → 2 × 6ns ≈ 12ns (读操作总线周转)

  写一像素: ADDSET + DATAST + 1 ≈ 7 HCLK ≈ 42ns
  读一像素: ADDSET + DATAST + BUSTURN + 1 ≈ 9 HCLK ≈ 54ns
```

### 10.3 写入速度

```
  理论写速率: 168M / 7 = 24M writes/s
  带宽: 24M × 16-bit = 48 MB/s
  屏幕需求: 800×480×60fps = 23M pixel/s
  → FSMC 带宽充裕，刷新 60fps 无压力
```

### 10.4 引脚占用

见 `02_完整引脚分配表.md` FSMC LCD 和触摸章节。共 24 pin：数据 D0~D15（PE7~15/PD14~15/PD0~1/PD8~10）、控制 NOE/NWE/NE4/A6（PD4~5/PG12/PF12）、背光 BL（PB15）。

### 10.5 注意事项

| 要点 | 说明 |
|------|------|
| **无中断需求** | FSMC 纯总线读写，不占用 NVIC 通道 |
| **与 SRAM 互斥** | 16 位数据总线被 LCD 独占，不能同时挂 FSMC SRAM |
| **CubeMX 配置** | FSMC NOR/SRAM Bank4, NE4=PG12, 16-bit 数据宽度, 禁用地址数据复用 |
| **读操作注意** | ILI9341 等 LCD 读像素不需要 FSMC 读，GRam 写即可 |
| **背光不占定时器** | PB15=LCD_BL 用 GPIO 高/低或简单 PWM，不需精密调光 |

## 十一、中断优先级分配总表

### 11.1 基础配置（只配一次）

```
  HAL_Init()
    └─ HAL_NVIC_SetPriorityGrouping(NVIC_PRIORITYGROUP_4)
         → 4 bit 全用于抢占优先级（0~15，数字越小越高）
         → 之后所有外设的 HAL_NVIC_SetPriority() 都在此框架内

  FreeRTOS 边界线:
    configLIBRARY_MAX_SYSCALL_INTERRUPT_PRIORITY = 5
    → 优先级 0~4: 不能调 FreeRTOS API（会破坏内核临界区）
    → 优先级 5~15: 可以调 FreeRTOS FromISR API
    → TIM6 Tick = 15 (最低优先级)
```

### 11.2 已配中断

| IRQ | 优先级 | 状态 | 来源 |
|-----|--------|------|------|
| TIM6_DAC | **15** | ✅ | FreeRTOS 1ms tick, HAL_InitTick(TICK_INT_PRIORITY=0x0F) |
| SysTick | 15 | HAL 内部 | 未使用（改 TIM6 替代），保留为备用 |
| PendSV/SVC | 15/0 | FreeRTOS | 内核中断，configKERNEL_INTERRUPT_PRIORITY |

### 11.3 待配中断（按 02_完整引脚分配表.md 外设顺序）

| # | 外设 | IRQn | 优先级 | DMA | 备注 |
|---|------|------|--------|-----|------|
| 1 | **GNSS USART2** | USART2_IRQn | **6** | DMA1_S5_CH4 | 9600 Rx, 可调 FromISR |
| 2 | **LoRa USART3** | USART3_IRQn | **6** | DMA1_S1_CH4 | 9600 Rx, 同上 |
| 3 | **5G USART6** | USART6_IRQn | **7** | DMA2_S2_CH5 | 115200 AT 指令 |
| 4 | **CAN1** | CAN1_RX0_IRQn | **7** | — | V1.1 实装 |
| 5 | **SDIO SD 卡** | SDIO_IRQn | **8** | DMA2_S3_CH4 | FatFs 写卡 |
| 6 | **I2C1 传感器** | I2C1_EV + ER | **9** | — | MPU6050+BME280 |
| 7 | **TIM3 PWM1/2** | — | — | — | **无需中断**，纯硬件 PWM 输出 |
| 8 | **TIM4 PWM3/4** | — | — | — | **无需中断**，同上 |
| 9 | **TIM13 蜂鸣器** | — | — | — | **无需中断**，同上 |
| 10 | **Touch GT911** | EXTI1_IRQn (PB1) | **12** | — | 触摸按下触发 |
| 11 | **按键 WK_UP** | EXTI0_IRQn (PA0) | **12** | — | WK_UP 按下 |
| 12 | **按键 KEY0~2** | EXTI2~4 (PE2~4) | **12** | — | 按键消抖 |
| 13 | **ADC1 电源** | ADC1_IRQn | **12** | DMA2_S0 | 低频采样 |
| — | **FSMC 屏幕** | — | — | — | **无需中断**，纯总线读写 |

### 11.4 优先级设计原则

| 原则 | 说明 |
|------|------|
| 6~7: 通信实时 | USART2/3/6 是数据入口，稍高优先级保证不丢帧 |
| 8~9: 低速 IO | SD 卡写入不紧急，I2C 传感器低频 |
| 10+: 无需中断 | PWM 纯硬件输出，不配中断 |
| 12: 人机交互 | 按键/触摸低频事件，低优先级 |
| 15: 系统时基 | TIM6 必须最低，否则破坏 FreeRTOS 调度 |

### 11.5 USART DMA IRQ 特殊要求

```
  USART2 DMA1_S5 与 USART2 必须同优先级（都是 6）
  USART3 DMA1_S1 与 USART3 必须同优先级（都是 6）
  原因: IDLE 中断触发时 HAL_UARTEx_RxEventCallback 里的 RxXferCount
        依赖 DMA 传输同步完成，若 DMA 优先级低于 USART 会导致数据不完整
```

### 12.2 每步操作模板

```
  1. 查 02_完整引脚分配表.md → 确认引脚、AF
  2. 查 08 节 二（外设时钟速查）→ 确认总线频率
  3. 查 08 节 十一（中断优先级总表）→ 确认优先级 + DMA 通道
  4. 写 BSP 初始化 → Bsp/Src/ 追加 XX_Init()
  5. 将 init 加入 BSP_Init() 调用链
  6. 写 HAL_MspInit 回调 → Core/Src/stm32f4xx_hal_msp.c
  7. 写驱动四函数 → Sensor_Driver/Src/ 新建
  8. 注册 → Sensor_Driver/Src/sensor_registry.c s_sensor_drivers[] 加一行
  9. 中断服务 → Core/Src/stm32f4xx_it.c 加 HAL_IRQHandler
  10. 编译 → 上板 → USART1 串口助手看自检输出
```

### 12.3 注意事项

| 项 | 说明 |
|----|------|
| **优先级分组只配一次** | HAL_Init 已执行，之后不再调 HAL_NVIC_SetPriorityGrouping |
| **TIM 无中断** | TIM3/4/13 只做硬件 PWM 输出，不使能 TIM_IT_UPDATE |
| **FSMC 无中断** | 屏幕纯寄存器读写，不占 NVIC |
| **DMA 与 USART 同优先级** | 见 11.5 |
| **每个外设独立 BSP 文件** | 不用全塞进 bsp_uart.c，按模块分文件 |
| **I2C1 两传感器同总线** | MPU6050(0x68) + BME280(0x76) 共用 PB6/PB7，一个 I2C 初始化搞定 |
| **验证方法** | 每步写完先跑 self_check → USART1 打印结果 → 通过再下一步 |
