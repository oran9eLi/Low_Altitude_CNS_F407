# 系统错误码表

本表定义 `App_ErrorCode_t`，用于 HMI 显示、CSV 日志、`CNS_State`、LoRa/网关上报和人工排查。

当前告警链路不再维护独立告警项。各模块统一构造 `App_AlarmMsg_t`，通过 `App_AlarmPost()` 投递到告警队列；告警任务按 `source + code + instance_id` 更新活动异常记录表 `App_AlarmRecord_t`。

关键边界：

- `ERR_OK` 只表示无异常，不用于 RAISE 消息。
- `App_AlarmMsg_t.header.code` 是主错误事实，payload 只保存现场上下文。
- 传感器驱动直接填写 `ERR_SENSOR_*` 错误码，注册表校验后透传，不再做 `Sensor_FaultReason_t` 到应用错误码的二次映射。
- `CNS_State.alarm.current_code`、HMI `alarm_code`、CSV `alarm_code` 和 LoRa 上报统一使用 `App_ErrorCode_t`。

## 错误码

| 错误码 | 名称 | 等级建议 | 触发条件 | 处理方式 |
| --- | --- | --- | --- | --- |
| 0x0000 | `ERR_OK` | 正常 | 无异常 | 无 |
| 0x0101 | `ERR_SYSTEM_SELF_CHECK_FAILED` | 一般/重要 | 未细分模块的自检失败 | 记录模块状态和异常上下文 |
| 0x0102 | `ERR_SYSTEM_HEAP_FAILED` | 严重 | FreeRTOS 堆内存分配失败 | 记录系统异常并停留现场 |
| 0x0103 | `ERR_SYSTEM_STACK_OVERFLOW` | 严重 | FreeRTOS 任务栈溢出 | 记录系统异常并停留现场 |
| 0x0104 | `ERR_SYSTEM_ALARM_QUEUE_FULL` | 重要 | 告警队列满 | 记录队列溢出，保留已有异常 |
| 0x0105 | `ERR_SYSTEM_ALARM_RECORD_FULL` | 严重 | 活动异常记录表满 | 优先保留高严重程度异常 |
| 0x0201 | `ERR_SENSOR_OFFLINE` | 重要 | 传感器离线或长时间无响应 | 页面提示、日志记录、尝试恢复 |
| 0x0202 | `ERR_SENSOR_NO_DATA` | 一般 | 当前没有可用数据 | 页面提示，可继续运行 |
| 0x0203 | `ERR_SENSOR_DATA_INVALID` | 一般/重要 | 数据校验失败、格式错误或数值不可信 | 丢弃该次数据并记录 |
| 0x0204 | `ERR_SENSOR_NO_FIX` | 一般 | GNSS 无有效定位 | 页面提示，可用模拟或降级数据 |
| 0x0205 | `ERR_SENSOR_INIT_FAILED` | 重要 | 传感器初始化失败 | 标记设备不可用并记录 |
| 0x0206 | `ERR_SENSOR_TIMEOUT` | 一般/重要 | 采样或通信超时 | 重试并记录细节 |
| 0x0301 | `ERR_COMM_LORA_OFFLINE` | 重要 | LoRa 链路离线 | 报警提示，尝试恢复链路 |
| 0x0302 | `ERR_COMM_TIMEOUT` | 重要 | 通信超时 | 记录并重试 |
| 0x0303 | `ERR_COMM_FRAME_INVALID` | 一般 | 通信帧格式或校验异常 | 丢弃该帧并记录 |
| 0x0401 | `ERR_DISPLAY_OFFLINE` | 一般 | 显示模块未就绪或离线 | 状态记录，后续重连 |
| 0x0402 | `ERR_DISPLAY_REFRESH_FAILED` | 重要 | 显示刷新失败 | 记录并尝试恢复 |
| 0x0501 | `ERR_STORAGE_NOT_READY` | 重要 | 存储介质未就绪 | 暂停写入，页面提示 |
| 0x0502 | `ERR_STORAGE_WRITE_FAILED` | 重要 | 存储写入失败 | 停止本次写入并记录 |
| 0x0503 | `ERR_LOG_WRITE_FAILED` | 重要 | CSV 日志写入失败 | 标记日志异常并记录 |
| 0x0601 | `ERR_PROTOCOL_PARSE_FAILED` | 一般 | 协议解析失败 | 丢弃消息并记录 |
| 0x0602 | `ERR_PROTOCOL_CHECKSUM_FAILED` | 一般 | 协议校验失败 | 丢弃消息并记录 |
| 0x0FFF | `ERR_UNKNOWN` | 严重 | 未分类异常 | 记录日志并提示排查 |

## 活动异常记录

活动异常的唯一 key：

```text
source + code + instance_id
```

示例：

```text
APP_MODULE_SENSOR + ERR_SENSOR_NO_FIX   + GNSS_SENSOR_ID
APP_MODULE_COMM   + ERR_COMM_TIMEOUT    + LORA_LINK_ID
APP_MODULE_SYSTEM + ERR_SYSTEM_HEAP_FAILED + 0
```

`App_AlarmRecord_t` 保存当前仍激活的异常：

```c
typedef struct
{
  uint8_t active;
  App_ModuleId_t source;
  App_AlarmSeverity_t severity;
  App_ErrorCode_t code;
  App_AlarmPayloadType_t payload_type;
  uint16_t instance_id;
  uint32_t first_tick_ms;
  uint32_t last_tick_ms;
  App_AlarmPayload_t payload;
} App_AlarmRecord_t;
```

## 传感器 payload

传感器异常 payload 使用 `App_AlarmSensorPayload_t`：

```c
typedef struct
{
  uint16_t sensor_id;
  uint16_t sensor_type;
  char name[APP_ALARM_NAME_LEN];
  uint32_t detail;
} App_AlarmSensorPayload_t;
```

`detail` 对应 `Sensor_Status_t.driver_error`，只作为调试细节，不替代 `header.code`。
