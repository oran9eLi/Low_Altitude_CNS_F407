# 系统错误码表

本表定义 `App_ErrorCode_t`，用于 HMI 显示、CSV 日志、LoRa/网关上报和人工排查。

`App_ErrorCode_t` 是对外稳定错误码；`App_AlarmId_t` 是内部告警项，用于告警激活、清除和 active mask。二者不要混用。告警事件统一构造为 `App_AlarmMsg_t` 后写入告警队列，告警任务消费队列并更新 `App_AlarmRecord_t`。

`App_AlarmRecord_t` 保存 `alarm_id + source + error_code + payload + detail`。`payload` 保存模块差异化异常上下文；`detail` 是从 payload 提取出的简化诊断值，不作为 HMI/日志/上报的主错误码。

传感器告警的 payload 使用 `App_AlarmSensorPayload_t`，包含 `device_id`、`sensor_type`、`severity`、`fault_reason` 和 `driver_error`。其中 `severity` 与 `fault_reason` 来自传感器域，`error_code` 仍由应用层映射为对外稳定的 `App_ErrorCode_t`。

`APP_ERROR_OK` 只表示无异常，不作为“使用默认错误码”的占位值。调用方已有具体标准错误码时使用 `App_AlarmRaise()`；调用方只能确定告警项、没有更具体错误码时使用 `App_AlarmRaiseDefault()`，由告警模块按 `App_AlarmId_t` 填充默认错误码。

`App_AlarmRaise()`、`App_AlarmRaiseDefault()` 和 `App_AlarmClear()` 是便捷封装，内部仍然构造 `App_AlarmMsg_t` 并调用 `App_AlarmPost()`。模块需要携带复杂上下文时，应使用 `App_AlarmBuildRaiseMsg()` 或 `App_AlarmBuildRaiseDefaultMsg()` 构造消息，再通过 `App_AlarmPost()` 投递。

| 错误码 | 名称 | 等级 | 触发条件 | 处理方式 | 状态 |
|---|---|---|---|---|---|
| 0x0000 | OK | 正常 | 无异常 | 无 | 待确认 |
| 0x0001 | ERR_SELF_CHECK_FAILED | 一般 | 未细分模块的自检失败 | 记录模块状态和 detail | 待确认 |
| 0x0002 | ERR_HEAP_FAILED | 严重 | FreeRTOS 堆内存分配失败 | 停止调度或进入安全态，记录日志 | 待确认 |
| 0x0003 | ERR_STACK_OVERFLOW | 严重 | FreeRTOS 任务栈溢出 | 停止调度或进入安全态，记录日志 | 待确认 |
| 0x1001 | ERR_HMI_OFFLINE | 一般 | HMI 无响应 | 串口提示，尝试重连 | 待确认 |
| 0x1101 | ERR_GNSS_OFFLINE | 一般 | GNSS 串口无数据 | 页面提示 | 待确认 |
| 0x1102 | ERR_GNSS_NO_FIX | 一般 | 定位无效 | 页面提示，可用模拟数据 | 待确认 |
| 0x1201 | ERR_SENSOR_I2C | 一般 | I2C 读取失败 | 重试，记录日志 | 待确认 |
| 0x1301 | ERR_LORA_OFFLINE | 重要 | LoRa 心跳超时 | 报警提示 | 待确认 |
| 0x1401 | ERR_SD_MOUNT | 重要 | SD 挂载失败 | 禁止写日志，页面提示 | 待确认 |
| 0x1501 | ERR_POWER_LOW | 重要 | 电量低于阈值 | 蜂鸣器报警 | 待确认 |
| 0x1601 | ERR_MOTOR_FAULT | 重要 | 电机无响应/过流预留 | 停机 | 待确认 |
| 0xFFFF | ERR_UNKNOWN | 严重 | 未分类异常 | 记录日志，提示排查 | 待确认 |

## 与告警项的关系

| 告警项 | 默认错误码 | 说明 |
|---|---|---|
| APP_ALARM_SELF_CHECK_FAILED | ERR_SELF_CHECK_FAILED | 未细分模块自检失败 |
| APP_ALARM_HMI_OFFLINE | ERR_HMI_OFFLINE | HMI/显示模块离线 |
| APP_ALARM_GNSS_OFFLINE | ERR_GNSS_OFFLINE | GNSS 离线 |
| APP_ALARM_GNSS_NO_FIX | ERR_GNSS_NO_FIX | GNSS 无有效定位 |
| APP_ALARM_SENSOR_FAULT | ERR_SENSOR_I2C | 传感器汇总异常，具体设备见 detail 或异常明细 |
| APP_ALARM_LORA_OFFLINE | ERR_LORA_OFFLINE | LoRa 链路离线 |
| APP_ALARM_STORAGE_FAILED | ERR_SD_MOUNT | 存储模块异常 |
| APP_ALARM_POWER_LOW | ERR_POWER_LOW | 电源异常 |
| APP_ALARM_MOTOR_FAULT | ERR_MOTOR_FAULT | 电机异常 |
| APP_ALARM_LOG_STORAGE_FAILED | ERR_SD_MOUNT | 日志写入或存储异常 |
| APP_ALARM_HEAP_FAILED | ERR_HEAP_FAILED | 堆内存失败 |
| APP_ALARM_STACK_OVERFLOW | ERR_STACK_OVERFLOW | 栈溢出 |

## 等级

| 等级 | 含义 |
|---|---|
| 正常 | 无需处理 |
| 一般 | 不影响主系统运行 |
| 重要 | 影响实验或安全，需要提示 |
| 严重 | 需要停机或人工处理 |
