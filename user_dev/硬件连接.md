# GNSS 硬件连接

> ATGM336H → STM32F407ZGTx 开发板

---

## 接线对照

```
ATGM336H (模块侧)              STM32F407 (开发板侧)
───────────────────────────────────────────────────
  TXD (黄/绿)       ────────►  PA3  (USART2_RX)
  RXD (白/蓝)       ◄────────  PA2  (USART2_TX)
  GND (黑)          ────────   GND
  VCC (红)          ────────   3.3V/5V (见供电选择)
  PPS (可选)        ────────   不接

  PE6 (使能)        ◄────────  模块供电控制 GPIO
```

## 引脚宏 (bsp_config.h)

| 信号 | 宏 | 值 |
|------|-----|-----|
| TX | `BSP_GNSS_TX_PORT/PIN` | GPIOA / GPIO_PIN_2 |
| RX | `BSP_GNSS_RX_PORT/PIN` | GPIOA / GPIO_PIN_3 |
| TX AF | `BSP_GNSS_TX_AF` | GPIO_AF7_USART2 |
| RX AF | `BSP_GNSS_RX_AF` | GPIO_AF7_USART2 |
| 使能 | `BSP_GNSS_PWR_PORT/PIN` | GPIOE / GPIO_PIN_6 |
| 波特率 | `BSP_GNSS_UART_BAUD` | 9600 |

---

## 供电选择

| 选择 | 电压 | 接法 |
|------|------|------|
| **推荐** | 3.3V | 直连开发板 3.3V 引脚 |
| 备选 | 5V | 部分模组支持宽压，确认无误再接 |

> ATGM336H 模块通常板载 LDO，输入 3.3V~5.5V 均可。

---

## 天线

- 有源天线 (3.3V 供电，模块板载天线供电电路)
- 天线需置于室外或窗边，室内 GPS 信号极弱
- 首次冷启动 (TTFF): 约 30~60 秒
- 热启动: 约 1~5 秒

---

## 通信参数

| 参数 | 值 |
|------|-----|
| 协议 | NMEA 0183 |
| 波特率 | 9600 bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 输出频率 | 1Hz (默认) |
| 语句前缀 | `$GNxxx` (GPS+BD 双模) |

---

## 验证步骤

1. 接好 VCC、GND、TXD→PA3、RXD→PA2
2. 下载阶段1固件
3. 打开调试串口 (USART1 115200)，应看到原始 NMEA 输出：
   ```
   $GNGGA,...
   $GNRMC,...
   $GNGSV,...
   ```
4. 如无输出，检查：
   - TXD↔PA3 方向是否正确
   - 天线是否接好
   - 波特率是否 9600
   - 模块指示灯是否闪烁 (1Hz = 定位成功)
