> 基于 `02_完整引脚分配表.md` + `Bsp/Inc/bsp_config.h`
> 目标设备: ATGM336H (NMEA 输出, 默认 9600bps)

## 一、开发路线图

```
阶段1              阶段2              阶段3              阶段4              阶段5
USART2 底层通  →  NMEA 帧解析    →  Sensor_Driver   →  模块自检        →  系统集成
(1-2天)          (2-3天)           注册挂载           (1天)              (2-4天)
                                    (1-2天)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 阶段 1：USART2 底层通 (BSP 层)
- [ ] 在 `stm32f4xx_hal_msp.c` 添加 `USART2` GPIO 初始化 (引用 `bsp_config.h`)
- [ ] 实现 `Bsp/Src/bsp_gnss.c` → USART2 初始化 + DMA/中断接收
- [ ] 调试控制台打印原始 NMEA 语句，验证接线无误

### 阶段 2：NMEA 帧解析 (Driver 层)
- [ ] 实现 NMEA 语句校验 (异或校验)
- [ ] 解析 **GGA** (定位信息) → 经纬度、高度、卫星数
- [ ] 解析 **RMC** (推荐最小数据) → 速度、航向、日期
- [ ] 解析 **GSV** (可见卫星) → 信噪比、仰角
- [ ] 单元测试覆盖

### 阶段 3：Sensor_Driver 注册挂载
- [ ] 实现 `Sensor_Driver_t` 函数指针表 (Init / SelfCheck / Read / Reset)
- [ ] 填入 `sensor_registry.c` 的 `s_sensor_drivers[]`
- [ ] 填充 `Sensor_Value_t` 联合体 (GeoValue: lat/lon/alt/speed/hdop)

### 阶段 4：模块自检
- [ ] 实现 GNSS SelfCheck → 检测数据有效性、卫星数门限
- [ ] 注册到 `App_Registry` → 上电自检流程
- [ ] 告警逻辑 → 超时无数据 → `ERR_SENSOR_GNSS_TIMEOUT`

### 阶段 5：系统集成联调
- [ ] 数据推送至 HMI 屏幕 (经纬度/速度显示)
- [ ] 数据写入 SD CSV 日志
- [ ] LoRa 透传远端
- [ ] 与 Debug_CnsDemo 联动

## 三、验收标准

| 阶段 | 验收口径 |
|------|---------|
| 阶段1 | 串口助手可见 `$GNGGA,$GNRMC...` 原始帧 |
| 阶段2 | 调试控制台打印解析后的经纬度、高度、速度 |
| 阶段3 | 调用 `ReadAll()` 能拿到 GNSS 数据 |
| 阶段4 | 上电自检报告 GNSS 正常/异常 |
| 阶段5 | HMI 显示 + CSV 写入 + LoRa 发送 工作正常 |

## 四、注意事项

1. **ATGM336H 上电冷启动需 30s+**，首次定位时间较长，不要误判超时
2. **GPS+BD 双模**默认开启，NMEA 语句前缀为 `$GNxxx`
3. **PPS 引脚可不接**，V1.0 暂不需要精确时间同步
4. **天线需接**，无天线室内很难定位
5. **USART2 波特率默认 9600**，与调试串口 115200 不同
6. **DMA 接收推荐 Idle 中断方式**，避免逐字节中断开销
