# 编码规范

> 延续项目现有风格，保持一致性。

---

## 命名规则

| 层级 | 前缀 | 示例 |
|------|------|------|
| BSP 层 | `BSP_` | `BSP_GNSS_Init()`, `BSP_UART_Init()` |
| 宏 | `BSP_xxx_` 大写下划线 | `BSP_GNSS_UART_BAUD` |
| Sensor 层 | `Sensor_` / `s_` (静态变量) | `SensorGnss_Init()`, `s_gnss_driver` |
| App 层 | `App_` | `App_TasksInit()`, `App_ModuleId_t` |
| Debug 层 | `DBG_` | `DBG_PRINT()`, `DBG_TEXT_PROJECT` |
| 类型定义 | `_t` 后缀 | `Sensor_Value_t`, `Nmea_Type_t` |
| 枚举值 | 层级 + 下划线大写 | `SENSOR_OK`, `ERR_SENSOR_GNSS_TIMEOUT` |

---

## 文件组织

```
Bsp/
  Inc/bsp_gnss.h      ← 对外接口声明 (HAL 无关)
  Src/bsp_gnss.c      ← 硬件初始化实现 (HAL 相关)
Sensor_Driver/
  Inc/sensor_nmea.h   ← NMEA 协议层 (纯 C, HAL无关)
  Inc/sensor_gnss.h   ← GNSS 驱动声明
  Src/sensor_gnss.c   ← 驱动实现 + NMEA 解析
```

**规则:**
- BSP 层管硬件 (HAL 调用在此层)
- Driver 层管协议 (不应 include HAL 头文件)
- 上层通过 Sensor_Driver 接口调用，不直接访问 BSP

## 注释风格

```c
/* ========== GNSS BSP 初始化 ========== */
/**
 * @brief  USART2 初始化，启用 DMA Idle 接收
 * @param  无
 * @retval HAL_OK / HAL_ERROR
 */
HAL_StatusTypeDef BSP_GNSS_Init(void);

/* 单行注释用 // 或 / * ... * /，保持一致 */
```

- 每个 `.c` 文件顶部加模块说明 (参考现有 `bsp_can.c`)
- 关键决策加 `⚠` 标记
- 调试代码用 `/***************DEBUG***************/` 包围块

## 代码风格

- **Tab 缩进** (现有代码为 tab, 4 空格宽度)
- 大括号 K&R 风格: `if (...) {`
- 条件编译调试块用 `/***************DEBUG***************/`
- `#define` 常量全大写，函数名 PascalCase 或 层级前缀_PascalCase
- 禁止 `printf` 直接使用，通过 `DBG_PRINT` / `DebugConsole` API
- 静态变量前缀 `s_`，全局变量前缀 `g_`

---

## 禁止事项

- ❌ 在 Driver 层直接调用 HAL 函数
- ❌ 硬编码引脚 (必须引用 `bsp_config.h`)
- ❌ 使用 `malloc/free` (FreeRTOS 用 `pvPortMalloc`)
- ❌ 在中断中调用 `printf`
- ❌ 裸 `while(1)` 阻塞等待 (FreeRTOS 用 `vTaskDelay`)
